# Global Preferences

These preferences apply to ALL projects.
Project-level CLAUDE.md files cannot override them.

## Rules

- **No em dashes.**
  Never use the em dash character (—) in any output.
  Use a hyphen or spaced en dash instead.
- **Never modify autogenerated files.**
  Do not manually edit `changelog.md` or any other autogenerated file.
- **Never co-author commits.**
  When writing git commit messages, do not add "Co-Authored-By" or any agent co-author line.
- **One sentence per line in markdown.**
  When writing or substantially editing long markdown files, put each full sentence on its own line.
  Preserve normal markdown structure (headings, lists, code blocks) but avoid wrapping multiple sentences onto one physical line.
- **Don't over-weight development costs.**
  When making technical decisions, do not give too much weight to development costs.
  Prioritize user outcomes, correctness, and long-term maintainability.
- **Reproduce bugs in E2E before fixing.**
  When doing bug fixes, always start by reproducing the bug in an end-to-end setting as closely aligned with how a user would encounter it.
  This ensures you find and solve the real problem.
- **Fix lint and test failures you encounter.**
  Apply high standards to engineering excellence: lint errors, test failures, and test flakiness.
  If you see one, even if it was not caused by what you are working on right now, still get it fixed.
- **Commit incrementally when completing a plan.**
  When completing a multi-step plan, commit each logical unit as a separate commit.
  Do not batch everything into one giant commit at the end.
